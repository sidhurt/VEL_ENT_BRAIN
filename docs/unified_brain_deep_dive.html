ThinkVelocity Unified Brain

Architecture & Mechanics Deep Dive

Confidential

The Unified Brain is not a static database; it is a living, dynamic knowledge graph that intelligently routes context to Language Models just-in-time (JIT). This document explains the ingenious mechanics beneath the surface—from graph topography and enterprise inheritance to the mathematical relevance engine that auto-evolves with user behaviour.

1

The Core Philosophy: Why a Graph Database?

Most LLM wrappers dump user preferences into a flat JSON file or a Vector Database (RAG). This fails at scale because context is inherently relationship-driven.

In the Unified Brain, we use Neo4j. Every employee, project, skill, and company rule is a Node. The connections between them are Edges.

By structuring data this way, we don't have to write complex SQL joins to figure out which rules apply to which user. We simply traverse the graph:

[User] -> [:MEMBER_OF] -> [Team] -> [:BELONGS_TO] -> [Organization] -> [:ENFORCES] -> [Policy]
2

Graph Seeding & Topography

The Personal Brain

Owned by the individual user. When an employee fills out their onboarding form, the system generates:

  • • A central User node (the anchor).
  • • Role and Domain nodes linked via [:HAS_ROLE] and [:EXPERT_IN].
  • • Project and Task nodes linked via [:WORKS_ON] and [:PERFORMS].
  • • Style nodes defining tone of voice.

The Company Brain

Owned by the admin. When an enterprise is established:

  • • Organization and Team nodes are created.
  • • Policy nodes (e.g., "No PII") are attached via [:ENFORCES].
  • The Magic Link: When a user is added, a [:MEMBER_OF] edge is drawn. Instantly, the user inherits every policy and enterprise project via graph traversal. Zero data duplication occurs.

The Memory Relevance Engine

When the Velocity Browser Extension intercepts a prompt, it makes a silent API call to /api/enhance. The Brain must decide exactly what 15 pieces of context to send back. It does this via a sophisticated scoring algorithm.

The 5-Dimensional Scoring Algorithm

+15x
Intent Matching (Keywords)

Base +20, plus +15 per matched word between prompt and node content.

+50
State Score (Lifecycle)

Active = 50. Recent = 20. Archived = 0.

2x
Frequency Score

Usage Count × 2. Rewards heavily traversed paths.

+15
Recency Score

Used in last 24h = +15. Used in last 7 days = +5.

+100
Identity Anchors

Roles & Domains get massive boosts so the LLM always knows who is speaking.

The Enterprise Governance Override

If an admin sets a company rule, it cannot be bypassed. The algorithm automatically assigns any node of type `Policy` a score of 10,000. This guarantees that enterprise compliance will win every single context assembly race and be injected into the final prompt.

4

Organic Auto-Evolution

There are no "interact" buttons. The graph learns silently based on the user's API requests.

Strengthening via Usage

Every time the engine selects a node, updateMemoryStates fires. It increments usageCount and updates lastUsed to right now.

Auto-Resurrection

If an archived project is specifically mentioned in a prompt (high Intent Score), it is pulled into the payload and instantly upgraded back to Active.

Organic Decay

decayMemories simulates time. Edge traversal drops to Recent after 7 days, and Archived after 30 days of no use.

The Archive Penalty

Archived memories receive a -1000 point penalty. They are actively hidden to save tokens and reduce LLM hallucinations, unless explicitly invoked.

Velocity Prompt Engine Integration

The Unified Brain operates as a headless, Just-In-Time (JIT) microservice.

1

Intercept

The user types in ChatGPT. The Velocity extension pauses the request.

2

Request

The extension silently POSTs to the Unified Brain API.

3

Assembly

The Brain runs the Scoring Algorithm, pulling the top 15 nodes + all 10,000-point Policies.

4

Delivery

The Brain returns a structured JSON ContextPack.

5

Injection

The extension formats that JSON into a hidden system prompt.

6

Execution

The massively enhanced prompt is forwarded to OpenAI. The user experiences zero latency, but receives an LLM response perfectly tailored to their identity, active projects, and legal boundaries.
