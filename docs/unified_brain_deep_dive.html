The Core Philosophy: Why a Graph?

Most LLM wrappers dump user preferences into a flat JSON file or a standard Vector Database (RAG). This fails at scale because context is inherently relationship-driven.

In the Unified Brain, we use Neo4j. Every employee, project, skill, and company rule is a Node. The connections between them are Edges.

No complex SQL joins. We simply traverse the graph:

[User] → [:MEMBER_OF] → [Team] → [:BELONGS_TO] → [Org] → [:ENFORCES] → [Policy]
2

Graph Seeding & Topography

The Personal Brain

Owned by the individual user. Automatically generated upon onboarding:

  • A central User anchor node.
  • Role and Domain linked via [:HAS_ROLE].
  • Project nodes linked via [:WORKS_ON].

The Company Brain

Owned by the admin. Establishes the guardrails:

  • Organization and Team hierarchies.
  • Policy nodes enforced globally.
  • The Magic Link Users inherit every policy and enterprise project via instantaneous graph traversal. Zero data duplication.

The Candidate Memory Layer (Shadow Graph)

The Unified Brain acts as an Information Extraction engine on the fly. When users type prompts, the LLM evaluates the prompt to identify unseen Projects, Tasks, or Domains.

  • Candidates are generated as floating nodes (:Candidate) with confidence scores.
  • Instead of immediately cluttering the graph with hallucinations, Candidates are stored in a pending state.
  • Repeated sightings increase the candidate's reinforcementCount.
  • Candidates can be explicitly promoted to formal memories (Project, Task) by the user, integrating them fully into the active context graph.
6

Enterprise Artifacts & Org Knowledge

Outputs generated by the LLM aren't just text; they are converted into Artifact nodes.

  • Saved with strict provenance metadata (generation model, brain version, prompt hash).
  • These Artifacts are connected back to the graph via relationship edges like [:REFERENCES], [:COMPLIES_WITH], and [:RELATES_TO].
  • Authoritative or Verified Artifacts can be pulled back into active context by the Engine. A user might receive a colleague's previously verified strategy document as JIT context.
